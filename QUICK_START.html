🧪 Advanced: Developer Features

Access via Browser Console

Open Developer Tools (F12) → Console tab and run commands:

View all registered users:

bloodDonationApp.getAllUsers()

Get blood group statistics:

bloodDonationApp.getBloodGroupStats()

Show all data in system:

bloodDonationApp.showAllData()

Get total number of donors:

bloodDonationApp.getTotalDonors()

💡 Tips & Tricks

💾 Data Storage: All data is saved in your browser's LocalStorage. Clearing browser data will delete all information.
🔄 Registration Validation: User IDs, emails, and phone numbers must be unique. Try registering a new user to add more donors to the system.
📱 Mobile Friendly: The application works on mobile devices too! All pages are responsive and mobile-optimized.
🔐 Security: This is a frontend-only application. For production use, implement a backend database.

🐛 Troubleshooting

❌ Login not working?

Make sure you're using the exact credentials:

  • Check for spaces in ID and password
  • Phone number must be exactly: 01765131604
  • Clear browser cache (Ctrl+Shift+Delete)

❌ Can't find registered users?

Registered users are stored in browser LocalStorage. Make sure:

  • You haven't cleared your browser's site data
  • You're on the same browser/device
  • Run bloodDonationApp.showAllData() in console to check

❌ Styles not loading?

Make sure all files are in the same folder and reload the page (Ctrl+F5)

❓ FAQ

Q: Can I use this without internet?

A: Yes! The application runs entirely in your browser. No internet connection needed after opening the page.

Q: Where is my data stored?

A: Your data is stored in the browser's LocalStorage. It persists even after closing the browser.

Q: Can I export my data?

A: Yes! Use bloodDonationApp.backupData() in the console to get a backup.

Q: How do I add the system to my website?

A: Simply copy all the files to your web server or integrate the code into your existing website.

Q: Can multiple users work simultaneously?

A: Each browser has its own LocalStorage, but you can add a backend database for multi-user support.
